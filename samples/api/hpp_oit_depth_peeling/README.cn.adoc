== 概览

本示例实现了一种使用深度剥离（depth peeling）的顺序无关透明（OIT）算法。场景为一枚圆环面（torus），其不透明度可在 UI 中调整，并能得到像素级精确的结果。算法基于 Cass Everitt 的 https://developer.download.nvidia.com/assets/gamedev/docs/OrderIndependentTransparency.pdf[原始论文]。

== 算法

OIT 算法由若干个“收集（gather）”阶段，随后一个“合成（combine）”阶段构成：

- 每个收集阶段渲染一层透明几何。第一阶段渲染第一层、第二阶段渲染第二层，以此类推；
- 第 N 层由每个像素从前到后排序后的第 N 个片段组成；
- 合成阶段在屏幕空间进行，从后往前合并图层图像，得到最终结果；
- 即使几何体相交，该算法也能产生像素级精确的结果；当几何层数超过收集阶段数时，最远层会被跳过，但视觉结果保持稳定（无闪烁）。

== 选项

[cols="2,4,4"]
|===
| 选项 | 描述 | 备注

| 摄像机自动旋转
| 启用摄像机自动旋转
|

| 背景灰度
| 指定背景颜色的灰度乘子（0.0 ~ 1.0）
|

| 物体 Alpha
| 指定透明物体的不透明度（0.0 ~ 1.0）
|

| 前层索引
| 要渲染的第一层（0 ~ 7）
|

| 后层索引
| 要渲染的最后一层（0 ~ 7）
| 不能小于前层索引
|===
